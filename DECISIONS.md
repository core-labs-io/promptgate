# Decisions

Non-obvious choices made while building promptgate, and why. Newest entries at the bottom.

- 2026-07-02: Scaffolded with `hatchling` + `src/` layout, zero required runtime dependencies. `tiktoken` is an extra (`[openai]`); `pytest`/`pytest-cov`/`ruff` live under `[dev]`. Per Design Principle 2.
- 2026-07-02: Heuristic token counter is `ceil(len(text) / 3)` chars-per-token, not the more common ~4. Real tokenizers average closer to 4 chars/token for English; dividing by 3 deliberately overestimates so the fallback never undercounts a budget (Design Principle 6). A fixed `MESSAGE_OVERHEAD_TOKENS = 4` per message approximates chat-format framing tokens that plain text counting misses, applied uniformly whether or not `tiktoken` is available.
- 2026-07-02: `safety.validate()`/`repair()` track tool-call validity with a single running set of `tool_call_id`s seen from assistant messages, rather than trying to model "turns" explicitly. A tool message is valid iff its `tool_call_id` was declared by *some* earlier-in-list assistant message. This is simple, handles parallel tool calls, and is exactly the check `repair()` needs to drop orphaned results after eviction.
- 2026-07-02: `Scrubber` never puts secret values in the returned `pii_map` — secrets get their own internal-only dedup table (`_ScrubState.secret_value_to_placeholder`) that is discarded when the call returns. This structurally enforces the one-way asymmetry (Design Principle 5) rather than relying on callers to avoid persisting it.
- 2026-07-02: `on_secret="raise"` raises `SecretFound` from inside the regex substitution callback, aborting the whole `scrub_text`/`scrub_messages` call on the *first* secret found (partial output is discarded by the caller). `Gate.explain()` always builds its own internal `Scrubber` with `on_secret="mask"` regardless of the configured mode, so a dry run never raises or writes to stderr, per Design Principle 7 ("explain() ... without mutating anything").
- 2026-07-02: Idempotency (scrubbing already-scrubbed text is a no-op) is enforced with an explicit `_is_placeholder()` guard in every masking callback that has a generic-enough pattern to accidentally re-match a `[[CATEGORY_n]]` placeholder (context-keyword assignments, connection strings, entropy detection, all PII patterns). The prefix/PEM patterns are narrow enough to be naturally idempotent without a guard.
- 2026-07-02: `window.apply()` force-keeps three sets of messages regardless of budget: pinned (by role or index), the last `keep_last` messages, and the single most-recent user message (even if `keep_last` would otherwise exclude it) — this is what guarantees "never evict the most recent user message" holds even at `keep_last=0`. If forced-keep content alone exceeds budget, or nothing gets force-kept at all (e.g. `keep_last=0` with no pin match on a non-user-terminated conversation), the last message is kept anyway and a warning is emitted, so `window.apply()` never returns `[]` for non-empty input.
- 2026-07-02: `Gate.rehydrate()`'s placeholder regex has three alternatives (`[[TOKEN]]`, `[TOKEN]`, bare `TOKEN`) tried in that order so double-bracket matches don't get eaten by the single-bracket or bare alternative first, and only substitutes tokens that are actually keys in the caller's `pii_map` — this bounds the "tolerate light model mangling" requirement without turning into an unbounded find-and-replace over arbitrary text.
- 2026-07-02: gitleaks would flag promptgate's own test fixtures (format-correct fake keys like `AKIA` + `X`*16 are needed to exercise the real detection regexes) as if they were live secrets, since it matches by pattern, not "does this look fake to a human." Added `.gitleaks.toml` allowlisting `tests/` and `benchmarks/generate.py` by path rather than trying to dodge the patterns with off-format fixtures, which would defeat the point of testing against them.
